
html이란 링크(연결)가 가능한 문서 언어
프로그래밍 언어가 아님 (=표시 언어, 마크업 언어: 사칙연산이 없다.)

표시만 할 줄 안다면 워드, 한글처럼 문서를 만들 수 있다.
html 렌더링용 언어로 발전하고 있다.

'멀티미디어' 기능을 수용하고부터 신문 -> 해리포터 신문으로 바뀜 (현재 우리가 아는 인터넷 뉴스·신문)
완성도가 높음 =웹브라우저를 이길 대체재는 없어보임

웹브라우저는 =문서 뷰어
워드, 한글등 문서 sw가 없던 시절엔 문서 만드는 방법이 이것이 유일했음 -> 현재도 유효한 경향

웹브라우저 최초의 기조: '무설치'
이에 맞게 어지간한 소프트웨어들도 다 이 '무설치' 에 가까운 형태로 진화되어 왔음 


문서는 꼭 '성격이 무엇인지' 정확하게 라벨링, 태깅을 해주는 편

웹브라우저의 강력한 장점: 기본값 셋팅이 뛰어남 
ex) <h1> 태그: 글씨 크기 설정, 볼드처리 적용 등
    문서 기반이다보니, 보편적으로 문서가 마땅히 가져야하는 특성에 어긋나는 부분은 거의없음
    
h1 {
    display: block;
    font-size: 2em;
    margin-block-start: 0.67em;
    margin-block-end: 0.67em;
    margin-inline-start: 0px;
    margin-inline-end: 0px;
    font-weight: bold;
    unicode-bidi: isolate;
}
전세계에 있는 거의 대부분의 디지털 도구들이 웹브라우저를 장착하고있다
html은 렌더러


객체 안의 객체:
상위 객체, 하위 객체
부모(parent), 자식(child)라 부른다
html: 편리함
객체: 커스텀 특화

API
Application Programming Interface
남이 만든 객체, 코드등 남이 만든 여러것들
소프트웨어를 만들 때 쓸법한 것들을 모아놓은 도구 상자

Tk= GUI API
os(파이썬 모듈 라이브러리)= API

마크업 언어는 '프로그래밍' 한다 말하기에는 좀...
마크업 언어가 프로그래밍 언어와 다른 점
1. 기획에 따른 답이 정확히 정해져있다.
    뉘앙스, 취향 없음
    충실하게 표시만 되면 된다.
    철저하게 표준을 따라야 한다. Standard
2. 학습 지점: 문서 만드는 것이 겁이 안 날 정도

html은 그 위상이 어마어마하다. 완성도가 아주 높음 (최고 수준)

xml이란 데이터 교환 확장자를 직접 해낸것
html은 사실 xml의 통신 가능한 버전의 아종
xml 원본 마크업이 따로 있었는데 html이 주객전도시킴

html은 태그가 정해져 있음 = 통신용 문서 전용(http...)
xml은 내맘대로 태그를 만들 수 있음 = 데이터 저장 일반

practics 객체 = assemble(조립) 작업을 직접해낸것
아무튼 assemble 작업은 마크업의 '규칙'만 준수하면 그만
